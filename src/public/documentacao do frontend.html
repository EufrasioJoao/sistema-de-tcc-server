🎯 1. Visão Geral

O frontend do Sistema de Gestão de TCCs é uma aplicação web moderna construída com Next.js 15 utilizando o novo App Router. A aplicação oferece uma interface intuitiva e responsiva para gerenciar Trabalhos de Conclusão de Curso (TCCs), incluindo monografias, dissertações e teses.

🎯 Objetivo: Fornecer uma interface web moderna, acessível e eficiente para o gerenciamento completo de TCCs em instituições de ensino superior.

Características Principais:

  • 🚀 Next.js 15 com App Router para performance otimizada
  • 🎨 Tailwind CSS para estilização moderna e responsiva
  • 🧩 shadcn/ui para componentes reutilizáveis
  • 📱 PWA (Progressive Web App) para experiência mobile nativa
  • 🔐 Autenticação JWT com middleware de proteção de rotas
  • 🌐 Internacionalização preparada para múltiplos idiomas
  • ♿ Acessibilidade seguindo padrões WCAG
  • 📊 Dashboards interativos com gráficos e relatórios

🔐 10. Autenticação e Autorização

Sistema de Autenticação:

🔑 Fluxo de Autenticação:
  1. Login via formulário com email/senha
  2. Servidor retorna JWT token
  3. Token armazenado em cookie httpOnly
  4. Middleware valida token em cada requisição
  5. Contexto da aplicação gerencia estado do usuário

Middleware de Proteção:

// middleware.ts export default async function middleware(req: NextRequest) { const path = req.nextUrl.pathname; const isProtectedRoute = protectedRoutes.some(route => path.startsWith(route) ); const cookie = (await cookies()).get("session")?.value; if (!cookie && isProtectedRoute) { return NextResponse.redirect(new URL("/auth/signin", req.nextUrl)); } const session = await decrypt(cookie); if (isProtectedRoute && !session?.data.userId) { return NextResponse.redirect(new URL("/auth/signin", req.nextUrl)); } return NextResponse.next(); }

Sistema de Permissões:

Papel Permissões Acesso
